# KeyFlow CLI 공식 릴리즈: 터미널에서 워크스페이스로 가장 우아하게 연결하는 방법

현대적인 작가, 개발자, 그리고 AI 에이전트를 위한 퍼블리싱 스튜디오 **[KeyFlow](https://keyflow.me)**에 공식 CLI 툴킷이 새롭게 추가되었습니다. 

이제 브라우저를 켜지 않고도 개발하시던 터미널 환경에서, 당신의 마크다운 초안을 곧바로 KeyFlow의 비공개 워크스페이스(Draft)로 쏠 수 있습니다. 

## 💡 개발 배경: The Agentic Workflow

AI가 대신 글의 뼈대를 잡아주고 정보를 집계해주는 "Agentic 시대"가 도래했습니다. 우리는 이 흐름에 맞춰 글쓰기 파이프라인 역시 완벽하게 자동화될 수 있어야 한다고 믿었습니다. 

**KeyFlow CLI**는 바로 이런 자동화 파이프라인의 **마지막 관문** 역할을 하기 위해 탄생했습니다.

1. 옵시디언(Obsidian), 노션(Notion) 등 다른 로컬 마크다운 환경에서 글을 작성합니다.
2. 터미널에서 `keyflow push` 한 번만 입력하면 즉시 클라우드 에디토리얼 스튜디오로 동기화됩니다.
3. CI/CD 봇이나 AI 스크립트가 스스로 글을 요약해 `keyflow push`를 트리거하여 자동으로 아카이빙합니다.

## ✨ 핵심 기능

- `keyflow login`: 한 번의 명령어 타이핑으로 브라우저를 통한 Google/GitHub OAuth 인증을 완료합니다. 
- `keyflow push <file>`: 업로드한 마크다운 파일의 최상단 `#` 태그를 파싱하여 세상에서 가장 편하게 제목을 지정합니다.
- `무인 모드(Headless)`: 서버나 자동화 파이프라인에서는 브라우저 인증이 불가합니다. 이 경우 `KEYFLOW_REFRESH_TOKEN` 환경 변수만 등록해두면 사람의 개입 없이 곧바로 작동합니다.
- `완벽한 보안 격리`: 전송된 모든 마크다운 파일은 무조건 **비공개 초안(Draft)** 폴더에 저장됩니다. 오타나 레이아웃이 깨진 상태로 대중에게 실수로 공개될 위험성이 차단됩니다.

## 🚀 30초 퀵 스타트

정말 30초면 충분합니다. 터미널을 열고 시작해보세요.

```bash
# 1. 글로벌 설치
npm install -g @keyflow-blog/cli

# 2. 브라우저 인증 
keyflow login

# 3. 작성해둔 마크다운 초안 올리기
keyflow push my-first-article.md
```

성공적으로 발행되면 고유한 웹 접속 주소가 터미널에 나타납니다. 브라우저에서 최종 컴파일된 아름다운 타이포그래피를 확인하시고, 공개(Publish) 버튼만 누르면 완전히 세상에 나갈 준비가 마무리됩니다.
